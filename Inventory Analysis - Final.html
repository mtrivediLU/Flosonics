Data Cleaning and Preparation

Objective: The primary objective was to clean and structure the raw data from the four provided files: Q3_shipping, Q3_sessions, Q3_device_logs, and Q3_customer_order to prepare them for accurate analysis.

Process:

Loading and Handling Missing Values: We loaded each dataset and handled missing values, replacing or removing invalid entries. Inconsistent data types were corrected.

Standardizing Date Formats and Timestamps: Dates were standardized to ensure consistency across all files (e.g., "YYYY-MM-DD").

Transformations for Key Fields: Columns such as StandardFlopatchID, StandardDateFirstUse, and StandardDateLastUse were transformed to ensure consistency.

Error Handling with Placeholders: Instead of removing rows, placeholders were used for missing or erroneous data to preserve context.

Data Cleaning Process:

Cleaning Invalid flopatch_id Entries: Removed rows with invalid values (e.g., NULL, -1, Error).

Handling Invalid date_delivered Entries: Removed rows with invalid or missing dates, like "1900-01-01" or NULL.

Bad Data Reference Table: Created a reference table for all rows removed during cleaning for future data quality improvements.

Excluding Pilot Sites: Pilot sites were filtered out, ensuring analysis focused on operational hospitals.

Final Data Sets:

  • Inventory Analysis with Transformed Data: All data, including errors.
  • Inventory Analysis with Clean Transformed Data: Only valid, clean data.
  • Inventory Analysis Excluding Pilot Sites: Clean data, excluding pilot sites.
  • Bad Data Table: Rows removed due to bad data.
